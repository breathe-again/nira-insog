# docs/

All planning and design documentation for Nira Insig. Read in order:

1. `01-vision-and-scope.md` — the *why* and the *what*. CEO-readable.
2. `02-system-architecture.md` — the *how*. Components and data flow.
3. `03-data-model.md` — entities, fields, relationships.
4. `04-functional-requirements.md` — what the system must do, capability by capability.
5. `05-tech-stack.md` — concrete technology choices.
6. `06-roadmap.md` — 12-week Phase 1 plan and Phase 2+ outlook.
7. `07-team-and-resources.md` — team composition, tooling, budget.
8. `08-insights-catalog.md` — every insight the system produces, with examples.

Conventions:

- Each doc is self-contained — you can hand any one of them to a stakeholder without context.
- Keep them up to date as the system evolves; treat the docs as living artifacts, not a launch deliverable that ages.
- Material changes get a PR with the rationale in the description.
